# FastQC Skill

FastQC is a quality control tool for high throughput sequence data. Modern high throughput sequencers can generate hundreds of millions of sequences in a single run. Before analysing this sequence to draw biological conclusions you should always perform some simple quality control checks to ensure that the raw data looks good and there are no problems or biases in your data which may affect how you can usefully use it.

Most sequencers will generate a QC report as part of their analysis pipeline, but this is usually only focused on identifying problems which were generated by the sequencer itself. FastQC aims to provide a QC report which can spot problems which originate either in the sequencer or in the starting library material.

## When to use it

Before cleaning â€” always:

Right after receiving your raw FASTQ files from the sequencer. You run it to diagnose the data:

* Is the quality good enough to proceed?
* Are there adapters present?
* Is there unexpected contamination?
* Is the GC content normal?

You don't make decisions blind â€” FastQC tells you what needs to be fixed before you touch anything.

2. After cleaning â€” to confirm

After running Trimmomatic, fastp, etc., you run FastQC again to verify that:

* Adapters were actually removed
* Quality scores improved
* You didn't lose too many reads


## FastQC Usage

Note that there is a docker (dnalinux/fastqc) that provides the fastqc command.


The docker command mounts your current directory to `/ftmp` inside the container. All file paths in the examples below are relative to `/ftmp/`, which maps to wherever you run the command from.




```bash
docker run --network=none -v $(pwd):/ftmp dnalinux/fastqc:latest /ftmp/query.fastq
```






# FastQC CLI Options Reference

## General
| Flag | Long Form | Description |
|------|-----------|-------------|
| `-h` | `--help` | Print help and exit |
| `-v` | `--version` | Print program version and exit |
| `-q` | `--quiet` | Suppress all progress messages on stdout; only report errors |

## Input / Format
| Flag | Long Form | Description |
|------|-----------|-------------|
| `-f` | `--format` | Force input format. Valid values: `bam`, `sam`, `bam_mapped`, `sam_mapped`, `fastq` |
| | `--casava` | Files come from raw CASAVA output. Files in the same sample group will be analysed as a set. Sequences with the filter flag set in the header will be excluded. Files must retain CASAVA naming conventions (gzipped, ending in `.gz`). |
| | `--nano` | Files come from nanopore sequences in fast5 format. Pass directories; all fast5 files within will be processed into a single output. |
| | `--nofilter` | When using `--casava`, do not remove reads flagged as poor quality. |

## Output
| Flag | Long Form | Description |
|------|-----------|-------------|
| `-o` | `--outdir` | Directory for all output files. **Must already exist** â€” the program will not create it. Defaults to the same directory as the input file. |
| | `--extract` | Unzip the output file after creation. If `--delete` is also set, the zip is removed after extraction. |
| | `--noextract` | Do not uncompress the output file after creation. Recommended for non-interactive/automated runs. |
| | `--svg` | Save report graphs in SVG format. |

## Analysis Behavior
| Flag | Long Form | Description |
|------|-----------|-------------|
| | `--nogroup` | Disable base grouping for reads >50bp. Shows data for every base. âš ď¸Ź **Warning:** Can crash on very long reads and produce extremely large plots. |
| | `--min_length` | Set an artificial lower limit on sequence length shown in the report. Useful for making comparable statistics across datasets with variable read lengths. |
| | `--dup_length` | Length to truncate sequences when defining duplicates (affects duplication and overrepresented sequences plots). Useful for long reads with miscalls or adapter dimers with UMI sequences. |
| `-k` | `--kmers` | Kmer length for the Kmer content module. Must be between `2` and `10`. Default: `7`. |

## Custom Reference Files
| Flag | Long Form | Description |
|------|-----------|-------------|
| `-c` | `--contaminants` | Path to a custom contaminants file. Format: `name[tab]sequence` per line. Lines starting with `#` are ignored. |
| `-a` | `--adapters` | Path to a custom adapters file. Format: `name[tab]sequence` per line. Lines starting with `#` are ignored. |
| `-l` | `--limits` | Path to a custom limits file for warn/error thresholds per module. Can also be used to exclude specific modules from output. Must mirror the default `limits.txt` in the Configuration folder. |

## Performance
| Flag | Long Form | Description |
|------|-----------|-------------|
| `-j` | `--java` | Full path to the Java binary to use. Defaults to `java` in system PATH. |
| `-t` | `--threads` | Number of files to process simultaneously. Each thread uses 250MB of memory. Max 6 threads on 32-bit machines. |
| | `--memory` | Base memory per file in MB. Default: `512`. Increase for files with very long sequences. |

## Temporary Files
| Flag | Long Form | Description |
|------|-----------|-------------|
| `-d` | `--dir` | Directory for temporary files used during report image generation. Defaults to the system temp directory. |
